Git Automated

Download Now

Once you download. Make sure you turn on you preferred settings on the settings page.

What Happens when you run these commands:

Well once you run the start command the process will run in the background. The scripts will look inside the GitAutomated directory and check if you place a repo project there. If you did, it will add and commit all changed and untracked files every 15 minutes ( only if you turned the setting on ). Then if you have commits staged for pushing, it will push to github or any other git version control every day between 12-1pm and 5-6pm. Refer to the Instructions at the bottom.

* Only run the commands in the scripts directory, or it will not work as intended.

Start Automation *

Your unique start command
$ nohup python gitStarted.py {{ session.userid }} {{ session.token }} &
  • Shell = $
  • Allows you to close the terminal = nohup
  • The automation = python gitStarted
  • Your Id = {{ session.userid }}
  • Token = {{ session.token }}
  • Background Process Invoke = &

Stop Automation *

$ python gitStop.py
  • Shell = $
  • Stop Background Process = python gitStarted

Automation Instructions

Start Automation

  1. Click Download Button-(only once).
  2. Unzip and place the directory where you would like. It contains another directory called scripts. You should not have to edit these file. If you do open one in your editor you will see comments explaining exactly what each line will do.
  3. Place any directories that are git repositories just inside the GitAutomated directory, but not in the scripts directory.
  4. Now in your terminal, change directory to the scripts directory that is inside of the GitAutomated directory using cd command.
  5. Once inside the scripts dir. Paste the Start Automation command there.

Stop Automation

  1. Change directories to the scripts directory and Paste the Stop Automation command there. You would generally do this once you are done of the day or want to stop automation.
  2. Note. If you done not stop this when you are done, it will keep running in the background even if you close your terminal and log out.
